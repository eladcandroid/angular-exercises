🎯 תרגיל: אפליקציית TODO עם Signals ב-Angular

תיאור כללי

תרגיל מקיף זה נועד לבחון את הידע שלך בנושא Angular Signals - מערכת ניהול המצב החדשה והמודרנית של Angular. זהו תרגיל ברמת ביניים-מתקדמת שמכסה את כל ה-APIs של Signals כולל signal, computed, effect, resource, httpResource, input, output, model, linkedSignal ועוד.

תרחיש: אתם בונים אפליקציית TODO מתקדמת לניהול משימות, עם תכונות כמו סינון, קטגוריות, סטטיסטיקות, ושמירה מקומית. האפליקציה תהיה מבוססת Signals בלבד, ללא שימוש ב-RxJS או ניהול מצב חיצוני.

⏱️ משך זמן משוער: 3-4 שעות 📊 רמת קושי: ביניים-מתקדמת 💻 טכנולוגיות: Angular 20, Signals API, TypeScript, Standalone Components

🎯 מטרות הלמידה

בסיום התרגיל תוכיח/י שאת/ה יודע/ת:

  • ✅ signal() - יצירה וניהול של מצב ריאקטיבי
  • ✅ computed() - חישוב ערכים נגזרים ממצב
  • ✅ effect() - טיפול בתופעות לוואי (side effects)
  • ✅ resource() - טעינת נתונים אסינכרונית
  • ✅ httpResource() - טעינת נתונים מ-API (ניסיוני)
  • ✅ input() - קבלת נתונים מקומפוננטת אב
  • ✅ output() - שליחת אירועים לקומפוננטת אב
  • ✅ model() - two-way binding עם signals
  • ✅ linkedSignal() - יצירת signals תלויים
  • ✅ viewChild() - גישה לאלמנטים בתבנית
  • ✅ contentChild() - גישה לתוכן מוקרן

📋 דרישות התרגיל

חלק א': מבנה הפרויקט

צור/י את הקומפוננטות הבאות (כולן standalone):

1. TodoAppComponent (קומפוננטת האב)

זוהי הקומפוננטה הראשית שמנהלת את כל המצב.

Signals שצריך להגדיר:

  • tasks - רשימת המשימות
  • filter - סינון לפי מצב (all/active/completed)
  • searchQuery - טקסט חיפוש
  • selectedCategory - קטגוריה נבחרת

Computed Signals:

  • filteredTasks - משימות מסוננות
  • activeTasks - משימות פעילות
  • completedTasks - משימות שהושלמו
  • categories - רשימת קטגוריות
  • stats - סטטיסטיקות

Effects:

  • שמירה אוטומטית ל-localStorage
  • לוג של שינויים

פונקציות:

  • addTask(task: Task)
  • updateTask(id: string, updates: Partial<Task>)
  • deleteTask(id: string)
  • toggleTask(id: string)
  • loadTasks()

2. TaskListComponent

קומפוננטה שמציגה רשימת משימות.

Inputs:

  • tasks = input.required<Task[]>()

Outputs:

  • taskToggle = output<string>()
  • taskDelete = output<string>()
  • taskEdit = output<Task>()

Computed:

  • sortedTasks - משימות ממוינות לפי תאריך יצירה

3. TaskItemComponent

קומפוננטה שמציגה משימה בודדת.

Inputs:

  • task = input.required<Task>()

Model (two-way binding):

  • completed = model<boolean>()

Outputs:

  • delete = output<void>()
  • edit = output<void>()

Computed:

  • isOverdue - בדיקה אם המשימה באיחור
  • priorityClass - מחלקת CSS לפי עדיפות

4. TaskFormComponent

קומפוננטה ליצירת/עריכת משימות.

Inputs:

  • editTask = input<Task | null>()
  • categories = input<string[]>(() => [])

Outputs:

  • save = output<Task>()
  • cancel = output<void>()

Signals:

  • title, description, category
  • priority, dueDate

Computed:

  • isValid - בדיקת תקינות הטופס

5. FilterBarComponent

קומפוננטה לסינון משימות.

Model (two-way binding):

  • filterType = model<FilterType>('all')
  • searchQuery = model<string>('')
  • selectedCategory = model<string | null>(null)

Inputs:

  • categories = input<string[]>(() => [])
  • stats = input.required<TaskStats>()

Computed:

  • hasActiveFilters - בדיקה אם יש סינונים פעילים

6. StatsComponent

קומפוננטה שמציגה סטטיסטיקות.

Inputs:

  • stats = input.required<TaskStats>()

Computed:

  • completionColor - צבע לפי אחוז השלמה
  • displayStats - סטטיסטיקות מעוצבות לתצוגה

📐 מודל הנתונים

Task Interface

כל משימה תכיל:

  • id: string
  • title: string
  • description: string
  • completed: boolean
  • category: string
  • priority: 'low' | 'medium' | 'high'
  • dueDate?: string
  • createdAt: Date

FilterType

TaskStats Interface

  • total: number
  • active: number
  • completed: number
  • completionRate: number

💾 דרישות טכניות

1. LocalStorage Persistence

השתמש/י ב-effect() כדי לשמור אוטומטית את המשימות ב-localStorage:

  • טעינה ראשונית של משימות קיימות
  • שמירה אוטומטית בכל שינוי
  • המרה נכונה של תאריכים

2. Search & Filter

השתמש/י ב-computed() ליצירת רשימה מסוננת:

  • סינון לפי מצב (all/active/completed)
  • חיפוש בכותרת ובתיאור
  • סינון לפי קטגוריה
  • צירוף כל הסינונים יחד

3. ViewChild & ContentChild

השתמש/י ב-viewChild() לגישה לאלמנטים:

  • פוקוס אוטומטי על שדה הטקסט בטופס
  • גישה לאלמנטים ספציפיים בתבנית

4. Linked Signals

השתמש/י ב-linkedSignal() ליצירת signals תלויים:

  • כשהקטגוריה משתנה, הסינון מתאפס
  • signals שתלויים בערכים אחרים

🎁 משימות בונוס (אופציונלי)

בונוס 1: Resource API

השתמש/י ב-resource() לטעינת משימות בצורה אסינכרונית עם טיפול ב-loading, error ו-value states.

בונוס 2: HttpResource

צור/י backend פשוט או השתמש/י ב-JSONPlaceholder וטען/י משימות מה-API באמצעות httpResource().

בונוס 3: Undo/Redo

הוסף/י פונקציונליות Undo/Redo עם signals: שמור/י היסטוריה של שינויים ואפשר/י חזרה וקדימה.

בונוס 4: Real-time Sync

השתמש/י ב-effect() לסנכרון אמת עם LocalStorage או Backend עם debounce.

בונוס 5: Optimistic Updates

הוסף/י עדכונים אופטימיים עם rollback במקרה של שגיאה בשרת.

בונוס 6: Drag & Drop

הוסף/י אפשרות לסדר מחדש משימות עם drag & drop וניהול ה-order ב-signal.

בונוס 7: Performance Monitoring

מדוד/י ביצועים של computed signals והצג/י את הזמן שלוקח לכל חישוב.

בונוס 8: Custom Utilities

צור/י פונקציות עזר מותאמות אישית כמו debouncedSignal או throttledSignal.

⚠️ Best Practices

שימו לב:

  • אל תשנו signals בתוך computed - computed צריך להיות pure function
  • השתמשו ב-effect רק ל-side effects - לא לחישובים
  • זהירות עם effect cleanup - נקו subscriptions ו-timers
  • השתמשו ב-OnPush Change Detection - עבור ביצועים טובים יותר
  • אל תעבירו signals כ-inputs - העבירו ערכים

💬 שאלות נפוצות

Q: האם אפשר להשתמש ב-RxJS?

A: רצוי להימנע. התרגיל מיועד ללמוד Signals בלבד.

Q: האם חייבים להשתמש ב-httpResource?

A: לא חובה, זה בונוס. httpResource עדיין ניסיוני.

Q: איך מטפלים ב-forms עם Signals?

A: השתמשו ב-model() ל-two-way binding או ב-signal() + events.

Q: האם אפשר להשתמש ב-NgRx Signals Store?

A: לא בתרגיל הזה. השתמשו רק ב-Signals API המובנה.

Q: מה ההבדל בין computed() ל-effect()?

A: computed() מחזיר ערך חדש, effect() מבצע side effects (כמו שמירה ב-localStorage).
